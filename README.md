# MealPreply

Квиз MealPreply хранится в папке `quiz/`.

## Как у нас все устроено

- `main` это staging-версия проекта
- staging открывается по адресу `https://natakister.github.io/mealpreply/`
- `prod` это production-версия проекта
- production открывается по адресу `https://quiz.mealpreply.com`

## Как вносить изменения

1. Все обычные изменения сначала попадают в `main`
2. После пуша в `main` staging обновляется автоматически
3. На staging проверяем, что все работает как нужно
4. Когда версия готова для пользователей, переносим `main` в `prod`
5. После обновления `prod` production обновляется автоматически в Dokploy

## Как деплоить

### Обновить staging

- добавить изменения в `main`
- запушить `main`
- подождать автодеплой GitHub Pages

### Обновить production

- убедиться, что staging работает правильно
- создать Pull Request из `main` в `prod`
- замержить Pull Request
- подождать автодеплой Dokploy

## Важно

- напрямую в `prod` не пушим
- `prod` нужен только для финальных, проверенных версий
- если нужно срочно исправить production, потом эти же изменения надо вернуть в `main`
